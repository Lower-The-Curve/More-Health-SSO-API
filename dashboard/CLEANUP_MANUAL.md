# Playbook: Post-Remix Dead Code Cleanup

---

## When to Use This

After a Remix migration is confirmed complete and the app is running locally. Use this to remove unused files and UI components that were left over from the legacy SPA.

---

## Prerequisites

- [ ] `pnpm --filter more-health-remix build` passes with no errors
- [ ] You are on a dedicated git branch (not main)
- [ ] No uncommitted changes in the working tree

---

## Hard Exclusions — Never Touch

| Path | Reason |
|---|---|
| `lib/api-zod/src/generated/` | Auto-generated by Orval |
| `lib/api-client-react/src/generated/` | Auto-generated by Orval |
| `scripts/src/extract-i18n.ts` | i18n tooling |
| `artifacts/more-health-remix/app/` | Production app — routes, layouts, auth |

---

## Step-by-Step Process

### 1. Identify zero-importer files

Run this scan on the directory you want to clean. Replace `PATH_TO_SCAN`:

```bash
for f in $(find PATH_TO_SCAN -name "*.ts" -o -name "*.tsx"); do
  base=$(basename "$f" | sed 's/\.[^.]*$//')
  hits=$(grep -rl "/${base}" artifacts/ --include="*.ts" --include="*.tsx" 2>/dev/null \
    | grep -v "node_modules" \
    | grep -v "${f}$")
  if [ -z "$hits" ]; then
    echo "ZERO: $f"
  fi
done
```

Common starting points:
- `artifacts/more-health/src/components/ui/`
- `artifacts/mockup-sandbox/src/components/ui/`
- `artifacts/more-health/src/components/mockups/`

### 2. Review the candidates

Before deleting, check each file:
- [ ] Is it referenced in any `index.ts` barrel export?
- [ ] Is it used as a string literal (route name, i18n key)?
- [ ] Is it the sole importer of another file? (If yes, flag that file too — it may become a zero-importer after deletion.)

Skip any file that is unclear. Only delete what is confirmed unused.

### 3. Delete in batches of max 10 files

Always use absolute paths:

```bash
rm /Users/intern/Documents/GitHub/More-Health-SSO-API/PATH/TO/file.tsx
```

### 4. Verify build after every batch

```bash
pnpm --filter more-health-remix build
```

If the build breaks: stop, restore the last deletion with `git checkout -- path/to/file.tsx`, and skip that file.

### 5. Re-run the scan after each batch

Deleting one file can turn its sole importer into a new zero-importer. Re-run Step 1 before starting the next batch.

### 6. Commit each passing batch

Keep commits small. Example message: `chore: remove unused UI primitives from more-health/src`

---

## Commands Reference

| Action | Command |
|---|---|
| Find zero-importer files | Batch scan script from Step 1 |
| Delete a file | `rm /absolute/path/to/file.tsx` |
| Verify build | `pnpm --filter more-health-remix build` |
| Restore a deleted file | `git checkout -- path/to/file.tsx` |
| Check a specific file's importers | `grep -rl "/BASENAME" artifacts/ --include="*.ts" --include="*.tsx"` |

---

## Validation

After each batch:
- `pnpm --filter more-health-remix build` exits with code 0
- No TypeScript errors related to deleted files

After the full cleanup:
- Re-run the batch scan on all cleaned directories
- Output should be empty (no `ZERO:` lines)

---

## Risks / Edge Cases

**Cascade orphaning** — deleting a file can silently make its sole importer a new zero-importer. Always re-run the scan after each batch.

**Dynamic imports in mockup-sandbox** — files under `mockup-sandbox/src/components/mockups/` are registered via dynamic `import()` in `src/.generated/mockup-components.ts`. They will appear as zero-importers if you search from the repo root (`.`) instead of from `artifacts/`. Always run grep from `artifacts/`.

**Legacy SPA build is broken by design** — `pnpm --filter more-health build` will always fail locally (requires Replit env vars). Use only `pnpm --filter more-health-remix build` for verification.
